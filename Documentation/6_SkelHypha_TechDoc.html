Documentation technique¶

Notes:

This script has been tested with the following dependencies (other versions may work):

  • Python 3.6.8, 3.7.2 and 3.7.3
  • numpy 1.16.2 and 1.16.3
  • skimage (scikit-image) 0.14.2 and 0.15.0
  • tifffile 0.15.1 and 2019.3.18

If the program takes several minutes to compute the skeleton of the slice 0 of the first image, check that the matrix of the binarized image is of type uint8 with the background at 0 value (not 255!). If it’s at 255, try switching on/off the line 114 which inverts the image.

SkelHypha.createSubImages(slices)[source]¶

Create 25 subimages made of 3x3 slices. The skeleton will be computed on 3x3 slices, but only the core slice will be kept for the final skeleton (to avoid skeleton errors when at the border of an image). Completely black slices (void) are added when necessary to create a 3x3 slices for each previously created slice.

Paramètres:slices (list[np.ndarray]) – the slices that will be used to create the subimages
Renvoie:a list of the subimages created and the processing time of the computing
Type renvoyé:(list[np.ndarray], int)
